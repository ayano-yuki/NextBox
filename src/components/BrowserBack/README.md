# ブラウザバックの禁止

`useDisabledBrowserBack` で実現している。
これは、**ブラウザの戻る操作（popstate）を検知し、戻り動作をキャンセルするためのカスタムフック** です。

## このフックがやっていること

1. 現在の URL を pushState で履歴に積む
   - popstate が確実に発火する状態を作る
2. popstate イベントを監視するリスナーを登録
   - ブラウザが「戻る」を試みた瞬間にフックされる
3. 戻ろうとしたらアラートを出し、再度 pushState で戻れないようにする
   - 実質「戻る」操作を無効化
4. コンポーネント unmount 時にリスナーを削除
   - メモリリーク防止

## 使い方

フックを `import` し、コンポーネント内部で実行する。
そうすると、そのコンポーネントがマウントされている間、**ブラウザの戻るボタンを押しても popstate が発火し、戻れなくなる。**

## シナリオ
- フォームやトランザクションの完了時
  - ユーザーがデータを送信したり、決済を完了したりした後、「戻る」ボタンを押して意図せず前の画面に戻り、誤って再送信してしまうのを防ぎたい場合
- 特定のアプリケーションフロー
  - ウェブアプリ内の特定のページ（例：モーダル画面、ウィザード形式のステップ）で、強制的に次のステップへ進ませたい、またはブラウザの履歴を無視させたい場合
